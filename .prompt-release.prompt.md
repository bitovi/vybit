# Release assistant prompt

Purpose
- Automate the common release workflow for this repository: bump package version, build artifacts, publish to npm, create a git commit and tag, and push to the remote.

Inputs
- `versionType` (string, required): one of `patch`, `minor`, `major`, or an explicit semver like `1.2.3`.
- `tag` (string, optional): git tag to create (defaults to `v<newVersion>`).
- `dryRun` (boolean, optional): when true, do not actually publish or push; show commands instead.
- `skipBuild` (boolean, optional): when true, skip the build step.

Behavior
1. Validate `versionType` and compute new version.
2. Update `package.json` version field to new version.
3. Run the build step (unless `skipBuild`).
4. Run `npm publish --access public` (unless `dryRun`).
5. Commit changes, create a git tag (unless `dryRun`).
6. Push commit and tag to remote (unless `dryRun`).

Output
- JSON with `oldVersion`, `newVersion`, `gitCommit`, `gitTag`, and `npmPublishResult` (or the commands that would have run if `dryRun`).

Example invocations
- Bump patch and publish:
  {
    "versionType": "patch",
    "dryRun": false
  }

- Prepare a release without publishing:
  {
    "versionType": "1.2.3",
    "dryRun": true
  }

Notes
- This prompt assumes the user has `npm` and `git` authenticated and configured.
- The prompt runs commands in the repository root.
- For CI usage, prefer setting `CI=true` and provide an npm token via environment.

Customization ideas
- Add changelog generation from commits.
- Support publishing to alternate registries via `npm_config_registry`.
- Add support for building only specific packages in a monorepo.
