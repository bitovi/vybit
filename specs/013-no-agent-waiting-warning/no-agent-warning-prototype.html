Spec 013 — No-Agent Warning

When the user has committed changes in the queue but no AI agent is currently calling implement_next_change, the changes will sit waiting indefinitely. These prototypes explore where and how to surface that state. Each option shows no-warning (agent present) and warning (no agent) side by side. Toggle the "agent polling" switch to simulate both states.

Recommended Option A Inline Footer Strip

A thin strip slides in above the queue footer when commits are waiting and no agent is watching. It shows a brief message and a one-click copy button for the exact prompt to give the agent. Low visual weight — it lives next to the queue counters it explains, and disappears the moment an agent connects or the queue drains.

Option B Committed Pill — Warning State + Tooltip

The "committed" count pill in the queue footer changes colour and gains a pulsing dot indicator when no agent is watching. Hovering reveals a tooltip with the exact instruction. Zero extra layout space — works entirely within the existing footer row.

Option C Body Callout Card

A callout card appears inside the panel body area — above the property chips or as a standalone message when no element is selected. This gives the most screen real-estate for a full explanation and a prominent copy button.
