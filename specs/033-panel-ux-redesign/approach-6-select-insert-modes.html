Approach 6 — Select / Insert Mode Toggle

A segmented control in the panel header switches between Select (element-first: click → inspect → edit) and Insert (action-first: choose what to add → place it). Both paths converge: after inserting, auto-switch to Select with the new element selected. The overlay toolbar provides contextual actions once an element is selected.

Insert supports both orderings: click Insert icon → click WHERE → choose WHAT; or click Insert icon → choose WHAT → click WHERE.

Select Mode

Element-first: click to inspect, toolbar for actions, panel shows design editor.

A · Select mode — no element selected

Default state. Mode toggle shows Select is active. Panel shows an empty state inviting the user to click an element. The page is in selection mode — hovering highlights elements.

Select Mode — Toolbar Pivot: Component / Canvas

Clicking a toolbar button changes what the panel shows — no tabs needed. In Select mode, 📦 means replace this element. Use Insert mode to add new elements.

C · Toolbar → Component button → replace this element

Clicking 📦 in the toolbar pivots the panel to the component browser. Pick a component to replace the selected element. No sub-popover needed — "insert nearby" is what Insert mode is for.

Insert Mode — Two Orderings

Users can go WHAT → WHERE (pick component, then place it) or WHERE → WHAT (click location, then choose what to insert). Both paths land in the same place.

E · Insert mode — action-first: choose WHAT

User switches to Insert mode. Panel shows the insertion options: Component (from Storybook) or Canvas (freeform draw). Clicking a component arms it; clicking Canvas starts the draw flow. Page shows subtle drop zone indicators.

Insert Mode — Location-First (WHERE → WHAT)

The user switches to Insert mode and clicks a location on the page before choosing what to insert. Drop zones appear on hover. After clicking a drop zone, the panel asks "what do you want to insert here?"

G · Insert mode — hovering to choose location first

User is in Insert mode but hasn't picked a component yet. Hovering the page shows drop zone indicators between elements. The panel's "What do you want to insert?" prompt is still visible. Click a drop zone to lock the location.

After Placement → Auto-Switch to Select

Inserting a component completes the Insert flow. The app auto-switches to Select mode with the new element selected, toolbar appears, and the design editor is ready for refinement.

I · Just placed → auto-switch to Select mode

The component was placed. Mode automatically flips to Select, the new element is highlighted, toolbar appears, and the panel shows the design editor. The user can immediately refine the new element's Tailwind classes without any extra clicks.
