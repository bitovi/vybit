Overlay Toolbar — Redesign Exploration

Focused on the Element Toolbar that floats near a selected element. Six buttons: Select (re-select), Select More (1+), Insert, Design (default active), Text (edit text), Replace. Plus a message input for adding context.

Exploring icon-only, text-only, and icon+label variants, with different arrangements of the toolbar buttons and message form.

1 · Icon-Only Toolbar

Compact icon buttons. Semantically grouped: page interaction (Select, Select More, Insert) | element editing (Design, Text, Replace). Message row positioned separately near the element.

1A · Buttons above, message below

Toolbar floats above the selected element. Message input sits below. Keeps the message close to the element while buttons stay out of the way. "Design" is active (teal highlight) by default.

3E-Insert · Insert active — toolbar only

When Insert is active the right side of the toolbar collapses to a single button that opens the component picker in the sidebar panel (not a dropdown). Select/1+ dim but stay present.

Naming pairs — the Select-mode and Insert-mode buttons open the same sidebar tab, so they should feel related:

Select mode Insert mode Notes
Swap Place Both short action verbs. "Swap this" vs "Place new." Clear intent.
Replace Place Keeps existing "Replace" label. Rhymes with Place.
Swap Add Most familiar words. Minimal cognitive load.
Components Components Same label in both modes. Context (Select vs Insert) clarifies intent. Simplest.
