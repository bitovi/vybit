3F · Toolbar + Panel Unified — Mode-Dependent Tabs

The toolbar and panel work together. A Select / Insert toggle in both the overlay toolbar and the panel header keeps them in sync. The panel tabs change with the active mode:
Select mode → tabs = Design | Replace
Insert mode → tab = Place
The Replace and Place tabs share the same layout: a canvas option + Storybook component list.

The toolbar also includes right-side action buttons after a separator that mirror panel tabs: Select mode shows Design (filled = active tab) · Text · Replace. Insert mode shows a single Place button. Clicking these buttons switches the corresponding panel tab (or activates text editing for Text — see specs/034-text-editing).

Select Mode — Design Tab Active

User has selected an element. The toolbar shows Select active (outline ring), Insert dimmed. The panel’s Select/Insert toggle mirrors the toolbar state. Tabs show Design and Replace, with Design active — displaying the familiar property chips.

Tab Behavior Summary

Quick reference for how toolbar mode maps to panel tabs.

Toolbar Mode Panel Tabs Notes
Select Design · Replace Design = property chips. Replace = canvas + components.
Toolbar right side: Design · Text · Replace
Insert Place Same layout as Replace (canvas + components). Single tab.
Toolbar right side: Place

Replace vs Place: Identical UI — the name changes to match the action semantics. In Select mode you're replacing an existing element. In Insert mode you're placing a new element.
