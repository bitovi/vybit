Approach 5 — Overlay Context Menu

Move action selection into the overlay via a right-click/long-press context menu on the selected element. The panel becomes a focused detail view that shows only the UI for whichever action was chosen. Actions are right where the user is looking — at the element itself.

A · Right-click context menu on selected element

User selects an element, then right-clicks (or long-presses, or clicks a "…" button in the toolbar). A context menu appears at the cursor with all available actions, plus an inline quick message field.

B · Panel as focused detail view

After choosing "Edit Design" from the context menu, the panel shows only the property editors — no tabs needed. A small breadcrumb shows what mode it's in. Selecting another action from the context menu replaces the panel content.
