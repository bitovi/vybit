Flexbox Visual Picker — Interactive Concepts

More interactive/spatial approaches to flex layout controls. These explore how to make the relationship between the two axes (main/cross) visually obvious, and how to combine justify + align into unified controls.

1 3×3 Alignment Grid

Combines justify-content (horizontal) and align-items (vertical) into a single 3×3 grid. Like Figma's alignment controls, but as a unified position picker. Click a cell to set both justify + align at once. The center dot shows current position.

3×3 grid maps to justify × align.
  • Columns → justify-content: start | center | end
  • Rows → align-items: start | center | end
  • Stretch + baseline handled via separate toggle or the "Dist" control
  • Space distribution (between/around/evenly) is a separate "Dist" control since it can't be represented in a 3×3 grid
Tradeoff: The 3×3 grid only covers start/center/end — not stretch, baseline, or space distributions. We add a "Distribute" segmented control to handle those. This feels intuitive for basic centering but requires a mental model shift for advanced layouts.
2 Integrated Playground

A miniature flex playground with a real preview area and a compact toolbar underneath. The preview shows child items arranged with the current settings. The toolbar has icon buttons organized by property group. Changes are reflected live in the preview pane.

All-in-one playground. The preview + toolbar pattern:
  • Top: live preview showing children arranged by current settings
  • Bottom toolbar: grouped icon buttons (dir | jc | ai)
  • Clicking a button updates the preview AND stages the change
  • Gap uses familiar ScaleScrubber chips below
  • Toolbar labels use CSS shorthand (jc, ai, dir)
Pros: Self-contained, preview gives immediate visual feedback, everything about flex layout is in one bounded area.

Cons: Toolbar gets crowded at 320px panel width. Preview is approximate.
3 Dual-Axis Rail Picker

A rectangular area with two visible axes. Teal = main axis (justify-content), orange = cross axis (align-items). Clickable hotspots along each axis let you pick alignment. The preview children update position in real-time. Direction toggle swaps which physical direction each axis controls.

Spatial/axis-aware. The dual rail makes the main/cross axis concept visible:
  • Teal rail + dots = main axis (justify-content)
  • Orange rail + dots = cross axis (align-items)
  • Click a dot to snap children to that position
  • Direction toggle rotates which physical direction = main
  • Tooltip on hover shows the Tailwind class name
Unique aspect: Makes the main/cross axis mental model explicit and visual. Users see the relationship between direction and alignment.

Concern: May be too abstract for users unfamiliar with axis concepts. Missing space-between/around/evenly distributions.
REC Design Recommendation
Recommended approach: Option E (Hybrid) from the first file — combined with the 3×3 grid for quick alignment.

Why:
  • The Direction mini-diagrams reuse the DirectionPicker visual pattern already in the codebase
  • The icon segmented controls for justify/align are compact and self-documenting (Figma proven)
  • The 3×3 grid (from this file) could be an optional "quick pick" for the most common start/center/end combos
  • All controls fit in ~130px vertical space within the 320px panel width
  • Gap uses the existing ScaleScrubber component — no new interaction to learn

Implementation plan:
  1. Create FlexControls modlet in panel/src/components/
  2. Sub-components: DirectionButtons, JustifyBar, AlignBar, WrapToggle
  3. Wire into Picker.tsx under the "Flexbox & Grid" section
  4. Reuse onHover/onLeave preview lifecycle from ScaleScrubber
Properties to support (priority order):
  • P0 — flex-direction (4 options), justify-content (7), align-items (5), gap (scalar)
  • P1 — flex-wrap (3 options)
  • P2 — align-content (9 options, only when wrap is active)
  • P3 — justify-items (4, rare in flex)

Interaction lifecycle (same as other panel controls):
  • Hover icon → preview class on the live element
  • Mouse leave → revert preview
  • Click → stage the class change
  • Works with the existing patchManager preview/revert/stage flow
