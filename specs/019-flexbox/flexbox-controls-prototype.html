Flexbox Controls — Panel UX Explorations

Design prototypes for parent flex property controls in the Tailwind Inspector panel. These explore different interaction models for direction, justify-content, align-items, align-content, wrap, and gap — the most commonly used parent flex utilities.

Figma-style approach. Compact, icon-heavy. Each property gets its own row with a small label and segmented icons.
  • Direction: arrow icons showing flow
  • Justify: bar-diagram icons
  • Align: vertical bar-diagram icons
  • Wrap: text labels (too few options for icons)
  • Gap: ScaleScrubber chip (reuse existing component)
Pros: Very compact, familiar from design tools, icons are self-documenting.

Cons: Icons may be too small at panel scale, requires custom SVGs, doesn't reuse existing ScaleScrubber pattern.

Hybrid approach — our recommendation. Combines the best of each:
  • Direction: spatial mini-diagrams (like DirectionPicker)
  • Justify/Align: icon segmented controls (Figma-style)
  • Wrap: tiny text toggle
  • Gap: familiar ScaleScrubber chip
Why this works:
  • Direction mini-diagrams reuse the DirectionPicker visual pattern
  • Segmented icons are compact + self-documenting
  • All controls fit in a single bordered card (~120px)
  • Consistent with the panel's existing visual language
  • Hover → preview, click → stage (same lifecycle)

justify-itemsjustify-items-start · justify-items-end · justify-items-center · justify-items-stretch
Priority order:
  • P0: flex-direction, justify-content, align-items, gap — nearly every flex container sets these
  • P1: flex-wrap — needed when content overflows
  • P2: align-content — only matters when wrap is on
  • P3: justify-items — rare for flexbox (more common in grid)
