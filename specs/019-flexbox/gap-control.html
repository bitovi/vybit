Gap Control — Prototype

A visual gap editor for flex containers. Supports gap (unified), gap-x (columns), and gap-y (rows) independently. The diagram makes the axis relationship explicit — orange = column gap, teal = row gap. Inspired by BoxModel ring layers and CornerModel slot pattern.

1 Interactive Panel Mock

The gap control sits below the justify/align chips in the Flexbox section. Click a gap chip to open the value scrubber. Toggle between unified (gap shorthand) and split (gap-x / gap-y) with the mode buttons. Hover the diagram zones to see axis highlighting.

Interaction model:
  • ↔↕ mode toggle — unified gap vs separate gap-x / gap-y
  • diagram hover — highlights the relevant zone(s), previews axis
  • chip click → opens the scrubber value list dropdown
  • chip drag ↔ → scrubs value directly (not shown in prototype)
  • Switching to "split" copies the current unified value to both axes
  • Switching to "unified" uses the gap-x value as the new gap

Axis coloring:
  • Orange = gap-x (column spacing ↔)
  • Teal = gap-y (row spacing ↕)
  • Unified mode uses teal for the combined value

3 Diagram Anatomy

The 3×3 CSS grid that forms the diagram. Items occupy the four corners. Gap zones fill the middle column (↔ gap-x) and middle row (↕ gap-y). The center intersection cell is decorative — hovering the whole diagram can later be used to set a unified gap shorthand.

Zone sizing: The grid column/row sizes for the gap zones scale with the current gap value, giving a proportional hint. For example:
gap-2 → 6px zone, gap-4 → 14px, gap-8 → 22px
This makes it easy to see at a glance whether the gap is tight or loose before reading the label.
