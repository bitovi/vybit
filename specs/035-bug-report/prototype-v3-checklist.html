035 · Bug Report Mode — v3 Checklist Layout

Three levels of information:
1. Primary events (click, nav, page-load, error, background) — always visible, checkbox always visible (unchecked by default)
2. Secondary summary rows (mutations, screenshot, network, logs) — always visible under their primary. Checkboxes appear when their parent primary is checked (all checked by default).
3. Detail expansion (actual log messages, screenshot images, DOM diffs) — future feature, designed now

Layout: TIME | CHECKBOX | CONTENT. Time shows relative deltas from the last primary event (e.g. −17s, −5s, 0s). Hover for absolute timestamp. Selection is free-form — check any primaries you want (non-contiguous OK). Shift+click for range select as a power-user shortcut.

Three-Level Information Model

How the three tiers of data relate.

Level What Visibility Checkbox Time Example
Primary User action or promoted event Always visible Always visible (unchecked by default) Relative delta from last event (hover for absolute) click on <button> in CommentForm
Secondary Consequence summary (per-type) Always visible (under primary) Indented under primary's, appears when parent is checked None (inherits from primary) 3 logs (2 errors)
Detail Actual data content Expand on click (future) N/A — inherits from secondary N/A ERR TypeError: Cannot read 'map'

Event Grouping Rules

Snapshot trigger Becomes Rule
click Primary Always — user-initiated
navigation Primary Always — route change or full-page nav
page-load Primary Always — initial or post-nav load
error Primary or Child Primary if outside grouping window; child if within 5s of a primary
mutation Secondary Groups under nearest preceding primary (up to 5s cap)
network-error Secondary Groups under nearest preceding primary
orphan (no primary within 5s) Primary background Synthetic grouping — polling, timer callbacks

Recording is always on. Background recording starts when the overlay initializes and runs in all modes. Bug Report mode provides the UI to browse and submit that recorded data. Grouping is visual only — the underlying flat snapshot array is unchanged.
