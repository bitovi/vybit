035 · Bug Report Mode

A new Bug Report mode joins Select and Insert in the panel header. Always-on background recording captures DOM mutations, screenshots, console logs, and network errors. Users switch to Bug Report mode, select a range of snapshots, describe the issue, and submit it as a bug-report patch to the AI agent.

The ModeToggle converts to icon-only buttons (with tooltips) to fit three modes without crowding. A custom bug icon is drawn in the same geometric, filled-path style as the existing Select and Insert icons.

Icon Gallery — Bug Icon in Context

The new bug icon matches the existing icon family: 16×16 viewBox, fill="currentColor", bold geometric silhouettes. Shown here at 18px for visibility alongside Select and Insert.

Mode Behavior Summary

How each mode maps to panel content and overlay behavior.

Mode Panel Content Overlay Behavior Output
Select Design · Replace tabs Crosshair cursor, element selection, highlights class-change, text-change, design, component-drop patches
Insert Place tab Crosshair cursor, drop zone indicators component-drop, design patches
Bug Report History list + description + Report button No crosshair, no selection — recording runs silently bug-report patch (screenshots, DOM, console, network)

Recording is always on. Background recording starts when the overlay initializes and runs in all modes. Bug Report mode just provides the UI to browse and submit that recorded data.
