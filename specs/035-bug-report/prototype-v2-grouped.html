035 · Bug Report Mode — Grouped Event Timeline

Events are structured as a primary → children tree. Primaries are user actions (clicks, navigation, page-load) or promoted errors/background updates. Children (mutations, screenshots, network, console logs) are grouped under their causal primary. Range selection operates on primaries — children are selected/deselectable individually within the range. Time displays on primaries only; children inherit context. Consecutive same-second primaries may suppress the repeated time.

Event Grouping Rules

How flat snapshot data maps to the visual primary → children tree.

Snapshot trigger Becomes Rule
click Primary Always — user-initiated
navigation Primary Always — route change or full-page nav
page-load Primary Always — initial or post-nav load
error Primary or Child Primary if outside any grouping window; child if within 5s of a primary
mutation Child Groups under nearest preceding primary (up to 5s cap)
network-error Child Groups under nearest preceding primary
orphan (no primary within 5s) Primary background Synthetic grouping — e.g., polling, timer callbacks

Recording is always on. Background recording starts when the overlay initializes and runs in all modes. Bug Report mode provides the UI to browse and submit that recorded data. Grouping is visual only — the underlying flat snapshot array is unchanged.
